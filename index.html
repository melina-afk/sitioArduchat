Arduchat: Juego de Damas con Comunicación IoT

Nuestro proyecto consiste en un juego de damas programado en Arduino que permite la interacción entre dos jugadores de manera física y digital. Cada movimiento realizado en alguno de los dos tableros se envía a través del protocolo MQTT utilizando el broker Mosquitto, lo que permite transmitir y visualizar en tiempo real la partida en un sitio web.

Además, los jugadores pueden enviarse mensajes durante la partida, los cuales también se publican en los tópicos de comunicación MQTT y se visualizan en el sitio web. Tanto los movimientos, los mensajes, como el estado general de la partida se registran automáticamente en una base de datos MySQL, permitiendo registrar y consultar el historial.

Este proyecto integra el uso de hardware, software y protocolos de red, demostrando la conexión entre el mundo físico y el digital mediante la Internet de las Cosas (IoT).

¿Qué es MQTT?

MQTT (Message Queuing Telemetry Transport) es un protocolo de comunicación ligero basado en el modelo publicador/suscriptor. Se utiliza ampliamente en sistemas de Internet de las Cosas (IoT) porque permite enviar datos entre dispositivos con bajo consumo de ancho de banda y gran eficiencia.

En nuestro proyecto, cada tablero físico de damas publica los movimientos y mensajes en diferentes tópicos MQTT, y la aplicación web se suscribe a esos tópicos para recibir los datos en tiempo real.

¿Para qué sirve el Broker Mosquitto?

Mosquitto es un broker MQTT, es decir, el servidor encargado de recibir, distribuir y gestionar los mensajes que se envían entre los dispositivos conectados.

Actúa como intermediario: los tableros Arduino publican información (movimientos o mensajes), Mosquitto la recibe y la reenvía a todos los clientes suscritos, como el sitio web. Esto permite que la comunicación sea en tiempo real, bidireccional y confiable.

¿Qué es el modelo MVC?

El Modelo-Vista-Controlador (MVC) es una arquitectura que separa la lógica del programa en tres partes: el Modelo (gestiona los datos y la base de datos), la Vista (interfaz que ve el usuario) y el Controlador (coordina la interacción entre ambos).

En nuestra aplicación web, este modelo se usa con PHP y MySQL para mantener un código ordenado y escalable. Así, cuando se reciben datos desde el broker MQTT, el controlador los procesa, el modelo los guarda y la vista los muestra actualizados.

Fat-Free Framework (F3)

Fat-Free Framework (F3) es un microframework PHP liviano y eficiente utilizado para desarrollar aplicaciones web de forma rápida y estructurada. Permite organizar el código bajo el modelo MVC (Modelo, Vista, Controlador), facilitando la separación lógica de la aplicación.

En nuestro proyecto, F3 fue fundamental para manejar las rutas, la conexión con la base de datos y el control de vistas dinámicas. Gracias a su simplicidad, logramos una integración limpia entre el backend (PHP) y la interfaz web, manteniendo un código claro y modular.

¿Cómo es el funcionamiento nuestro proyecto?

El sistema conecta dos pantallas OLED donde se dibuja el tablero de damas programado en Arduino. Cada movimiento realizado por un jugador se envía mediante el protocolo MQTT al broker local (Mosquitto). Desde allí, la aplicación web recibe en el tópico para movimientos y refleja esos movimientos en tiempo real, permitiendo además el envío de mensajes entre ambos jugadores.

1. Movimiento detectado

El Arduino capta el movimiento de una ficha en el tablero físico.

2. Envío al broker MQTT

El mensaje se publica en un tópico (por ejemplo, arduino/movimientos).

3. Recepción web


El listener suscripto al tópico de movimientos hace que el sitio web muestre el movimiento en el tablero virtual.

4. Visualización en tiempo real

Quienes no esten jugando podrán seguir el proceso de la partida
